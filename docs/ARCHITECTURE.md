# Architecture

ClipVault is a four-layer Clean Architecture: project references point inward toward `Domain`, which
references nothing. This document maps *where* code lives and *how* it composes; for *why* a design
was chosen see the [ADRs](./ADR_INDEX.md), and for the security rationale see [SECURITY.md](../SECURITY.md).

## Layers

```text
ClipVault.App             WinUI 3 — windows, tray, hotkey, composition root   (net10.0-windows)
  │  references ▼ Infrastructure, Application, Domain
ClipVault.Infrastructure  Adapters — SQLite, crypto, WinRT/Win32, hosted svcs (net10.0-windows)
  │  references ▼ Application, Domain
ClipVault.Application     Use cases — capture pipeline, queries, retention     (net10.0)
  │  references ▼ Domain
ClipVault.Domain          Entities, value objects, ports                       (net10.0, BCL only)
```

The dependency rule: a layer may reference only the layers below it. `Domain` references nothing;
`App` is the composition root and references all three.

- **Domain** — `src/ClipVault.Domain`. Framework-free core: entities (`ClipboardEntry`), value objects
  (`ClipContent`, `ClipboardSnapshot`, `EntryId`, `ContentHash`, `ClipboardPrivacySignals`,
  `SourceApplication`), the `ClipContentType` enum, and the port interfaces every outer layer
  implements (`IClipboardHistoryRepository`, `IEncryptionService`, `IClock`, `IClipboardMonitor`,
  `IClipboardWriter`). No package references (BCL only). Folders: `Abstractions/`, `Entities/`,
  `Policies/`, `ValueObjects/`.
- **Application** — `src/ClipVault.Application`. Use cases and the capture pipeline: `CaptureGate`
  runs ordered `ICaptureRule`s (rejection-first), content classifiers
  (`IClipboardContentClassifier`) flag secrets, `HistoryQueryService` answers reads, and retention
  trims history. Depends on Domain only. Folders: `Capture/` (`Rules/`, `Classifiers/`), `History/`,
  `Retention/`, `Messages/`, `Settings/`, `Clipboard/`, `Insights/`, `Abstractions/`,
  `DependencyInjection/`.
- **Infrastructure** — `src/ClipVault.Infrastructure`. Windows adapters that implement the Domain
  ports: `ChaCha20Poly1305EncryptionService`, the key vaults
  (`DpapiKeyVault`/`PassphraseKeyVault`/`ResolvedKeyVault`/`EphemeralKeyVault` behind `IKeyVault`,
  with `KeyProtector`), `SqliteClipboardHistoryRepository` / `InMemoryClipboardHistoryRepository`,
  WinRT/Win32 clipboard, and the `IHostedService`s. `net10.0-windows` (WinRT/DPAPI/CsWin32). Folders:
  `Security/`, `Persistence/`, `Clipboard/`, `Hosting/`, `Time/`, `Settings/`, `Startup/`,
  `DependencyInjection/`.
- **App** — `src/ClipVault.App`. WinUI 3 windows + view models, tray/hotkey (`Platform/`),
  localization (`Localization/`), and the composition root. `net10.0-windows`, unpackaged `WinExe`.
  Folders: `ViewModels/`, `Services/`, `Platform/`, `Localization/` (`Strings/`), `Assets/`,
  `Properties/`.

## Composition (DI)

The host is a `Microsoft.Extensions.Hosting` builder created in `App.StartupAsync`
(`App.xaml.cs`, via `Host.CreateApplicationBuilder`). `ConfigureClipVault`
(`HostBuilderExtensions.cs`) wires the three registration extensions — one per layer:
`AddPresentation()` (App), `AddApplication()`
(`ApplicationServiceCollectionExtensions`), and `AddInfrastructure(options)`
(`InfrastructureServiceCollectionExtensions`). Storage settings select the backend and key vault at
startup: volatile mode → `InMemoryClipboardHistoryRepository` + `EphemeralKeyVault`; encrypted-disk
mode → `SqliteClipboardHistoryRepository` + DPAPI / passphrase / Windows Hello vault. Background work
runs as hosted services (`ClipboardMonitorHostedService`, `RetentionHostedService`).

## Cross-cutting patterns

- **Messaging** — `CommunityToolkit.Mvvm` `IMessenger` (`WeakReferenceMessenger.Default`); e.g.
  `HistoryChangedMessage`. No direct view-model-to-view-model calls.
- **UI thread** — `IUiDispatcher` (Application) wraps the WinUI `DispatcherQueue` (`UiDispatcher` in
  App) so inner layers marshal to the UI thread without referencing XAML.
- **Localization** — the `{loc:Str Key=…}` XAML markup extension (`StrExtension`) resolves through
  `ILocalizationService` against embedded per-language JSON (`Localization/Strings/{en,ja,zh-Hans}.json`);
  the language is fixed for the process lifetime. Not `.resw` (see [ADR-0007](./adr/0007-custom-json-localization.md)).
- **Win32 interop** — generated by CsWin32 (`NativeMethods.txt`); `AllowUnsafeBlocks` is scoped to
  Infrastructure and App.

## Where do I add X?

| I want to add…                | Layer          | Typical location                                  | Notes |
| ----------------------------- | -------------- | ------------------------------------------------- | ----- |
| A new secret/classifier       | Application    | `Capture/Classifiers/` + register in `AddApplication` | Implements `IClipboardContentClassifier`. |
| A new capture rule            | Application    | `Capture/Rules/` + register in `AddApplication`   | Registration order = evaluation order (rejection-first). |
| A new storage backend         | Infrastructure | `Persistence/` + wire in `AddInfrastructure`      | Implements `IClipboardHistoryRepository`. |
| A new clipboard content type  | Domain + Infra | `ValueObjects/ClipContentType` + `Clipboard/`     | Extend the enum, then handle it in capture/write. |
| A UI window or page           | App            | `*.xaml` + view model in `ViewModels/`            | Register presentation services in `AddPresentation`. |
| A new persisted setting       | Application + Infra | `Settings/` (both layers)                    | Model in Application, persistence in Infrastructure. |
| A background job              | Infrastructure | `Hosting/` + `AddHostedService` in `AddInfrastructure` | Implement `IHostedService` / `BackgroundService`. |
| A new localized string        | App            | `Localization/Strings/{en,ja,zh-Hans}.json`       | Reference via `{loc:Str Key=…}`; add to every language. |
